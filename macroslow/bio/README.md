## Executive Summary

The **bio.md** living memory system evolves into a powerful, executable format by adopting **MAML (Markdown as Medium Language)** syntax as `.maml.md`. This creates a self-contained, versioned, schema-validated "biological ledger" that unifies identity (SOUL), capabilities (SKILLS), long-term knowledge (MEMORY), runtime state (CONTEXT), and conversation history (SESSION TREE) into one portable artifact.

MAML enhances bio.md with:
- **YAML Front Matter** for machine-parsable metadata, permissions, and bioinformatics DP scoring.
- **Structured Markdown Sections** that double as executable prompts and skill definitions.
- **Code_Blocks** for heartbeat processing, DP alignment, memory consolidation, and skill refinement.
- **Input/Output Schemas** and immutable **History** for auditability and reflective agent loops.
- **MCP Compatibility** for tool orchestration in Hermes-style (persistent memory) or OpenClaw-style (gateway) harnesses.

This hybrid format turns bio.md into the **single source of truth** for local, offline agents running on Ollama (or any LLM), while enabling advanced bioinformatics-inspired dynamic programming (DP) for optimal alignment, compression, and evolution of the agent's "mind."

It works seamlessly with local-only setups on Debian/Linux, supports small models (e.g., Qwen2.5 0.5B), and scales through pruning and JSON-backed DP layers. Agents become truly persistent, self-improving entities that survive reboots and evolve via heartbeats.

### Primary Use Cases for MAML-Enhanced bio.md
- **Living System Prompt**: Inject the full `.maml.md` (or key sections) as the system message for every Ollama interaction.
- **Self-Updating Ledger**: Heartbeats trigger DP-aligned updates to MEMORY and SKILLS sections.
- **Skill Orchestration**: Embed and execute reusable skills directly within the bio.maml.md file.
- **Memory Evolution**: Use DP algorithms (Needleman-Wunsch style) for optimal consolidation of heartbeats into curated knowledge.
- **Session Management**: Branchable SESSION TREE with optimal path reconstruction.
- **Inter-Harness Portability**: Share bio.maml.md files between local Ollama harnesses, Hermes memory plugins, or OpenClaw gateways.

### Strategic Benefits
- **True Persistence & Self-Awareness**: Survives restarts; evolves via automated consolidation.
- **Efficiency with Small Models**: DP compression and structured sections minimize context bloat.
- **Executable & Auditable**: Code blocks + History enable safe, traceable agent loops.
- **Bioinformatics Fidelity**: Treats memory/skills as "sequences" for alignment, folding, and binding.
- **Zero-Setup Bootstrap**: Harness auto-creates the file with sensible defaults.
- **Advanced Extensions**: Multi-agent coordination, cron-based "dream" cycles, inotify real-time updates.

### Core File Anatomy
Every `bio.maml.md` follows a dual-layer structure optimized for human readability, LLM prompting, and automated harness processing:

1. **YAML Front Matter** (`---` delimited): Metadata, DP scores, version, permissions.
2. **Markdown Body**: Semantic sections (`## SOUL`, `## SKILLS`, etc.) with embedded code blocks and JSON.
